## Priority Issues (all P3)

**[P3] Goal row gives no orientation cue for first-timers**
- **What**: Warm-brown tint on rows 1 and 9 is the only visual cue for where each player is headed. No text, no arrow, no label. HowToPlay says "SOUTH starts on row 1" but the board has no row numbers.
- **Why it matters**: Jordan sees two pawns, doesn't know which end to aim for. The tint is too subtle to read as "your goal" without prior knowledge.
- **Fix**: On the first turn, show a one-word overlay on the goal-row frame ("Goal →") fading out after the first move. Add aria-label to goal-row cells for screen readers.
- **Suggested command**: $impeccable onboard

**[P3] Wall mode reset is silent — no visual cue on mode switch**
- **What**: After placing a wall, handlePlaceWall resets actionMode to 'move'. Hint text updates but no animation on the toggle switch.
- **Why it matters**: Jordan places a wall, expects to place another, taps a grid line, nothing happens. No feedback on why the mode changed.
- **Fix**: Brief pulse/ring animation on the "Move" ToggleBtn when mode resets. 200ms is enough.
- **Suggested command**: $impeccable animate

**[P3] Wall hit-strip layer appears without ARIA notification**
- **What**: Switching to wall mode mounts 64 hit-strip buttons. No aria-live region announces this. Sam has no indication that 64 new interactive elements appeared.
- **Fix**: Add sr-only aria-live span: {actionMode === 'wall' ? 'Wall placement mode. Select a grid line.' : ''}
- **Suggested command**: $impeccable audit
