### Anti-Patterns Verdict

**LLM assessment:** Not obviously AI-generated. The 60/30/10 white/ink/brown palette holds across every screen. No gradient text, no glassmorphism, no hero metric template. Board-first philosophy is genuine. Two risk areas: numbered rule cards (01-04 as kicker eyebrows) and Fraunces appearing as the display font across many AI-generated UIs.

**Deterministic scan:** 2 findings in src/styles.css — both overused-font warnings for Fraunces (line 2 import, line 99 font-family). No findings in routes or components.

### Overall Impression

Coherent visual language, board earns its prominence. Blocked by three issues from feeling shipped: header still says "TanStack Start," About page is template boilerplate, and a bot API failure permanently locks the game with no recovery. Fix those and the foundation is solid.

### What's Working

1. **Board design:** Goal-row tinting, pawn gradients, real-time wall preview (brown=valid, red=invalid) — legible without text overlays.
2. **Win meter:** Dual-label, aria-labeled, transitions smoothly, dims during analysis.
3. **Home page:** Single island-shell, two actions, one purpose. No competing chrome.

### Priority Issues

**[P1] Bot API failure permanently blocks the game**
- `reset()` does not call `setThinking(false)`. Bot API failure leaves `thinking=true`, `interactive=false`. Player is stuck indefinitely.
- Fix: Add `setThinking(false)` to `reset()`. In catch block, show inline error beneath status chip. Move `setThinking(false)` outside the `if (!cancelled)` guard.
- Command: $impeccable harden

**[P1] Header and nav are template boilerplate**
- Brand chip reads "TanStack Start." Nav has TanStack Docs link and social icons to TanStack X/GitHub.
- Fix: Replace brand chip with "Wall Chess." Remove/replace TanStack-specific links.
- Command: $impeccable clarify

**[P1] About page is template placeholder text**
- Heading: "A small starter with room to grow." Body describes TanStack Start framework, not Wall Chess.
- Fix: Rewrite as Wall Chess product story and ML bot context.
- Command: $impeccable clarify

**[P2] Wall placement non-functional on touch screens**
- Hit-strip buttons use onMouseEnter/onMouseLeave. Touch events don't fire these. Preview never shows before tap.
- Fix: Replace with onPointerEnter/onPointerLeave, or add touch-first two-tap flow.
- Command: $impeccable adapt

**[P2] rise-in animation has no prefers-reduced-motion fallback**
- animation-fill-mode: both gates opacity:0 content. No @media (prefers-reduced-motion) alternative.
- Fix: Add prefers-reduced-motion rule resetting animation, opacity, transform.
- Command: $impeccable animate

**[P2] border-radius: 32px on island-shell cards**
- Home hero and HowToPlay modal use rounded-[2rem] = 32px, crossing the absolute ban threshold. DESIGN.md spec is 28px.
- Fix: Change rounded-[2rem] to rounded-[1.75rem] (28px).
- Command: $impeccable polish

### Persona Red Flags

**Jordan (First-Timer):** Cannot discover wall placement — "Hover a line between squares" but no visible lines exist. SOUTH/NORTH terminology unclear without context of which row is 1.

**Sam (Accessibility):** Focus not trapped in HowToPlay modal. ToggleBtn lacks aria-pressed. No ARIA live region for "Bot thinking" state change.

**Casey (Mobile):** Wall placement non-functional on touch. Player card ordering creates long scroll. "Change mode" link at page bottom is out of thumb zone.

### Minor Observations

- `<h2>` wraps brand chip in Header — inverted heading hierarchy (h2 before h1).
- Win meter shows raw numbers (47/53) with no % symbol.
- `reset()` skips `setThinking(false)` — race condition if reset mid-bot-turn.
- Graph page has no explanation for non-developer users.
- Numbered rule cards (01-04) look like AI section scaffolding despite the sequence being genuine.
